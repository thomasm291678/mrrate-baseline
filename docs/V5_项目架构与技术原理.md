# V5 多模态 MRI 放射学报告生成系统 — 架构与技术原理

> **模型**：Qwen2.5-3B-Instruct + MRRCNN 编码器  
> **任务**：多模态 MRI 影像（T1/T2/FLAIR）→ 放射学报告自动生成（37 类 Clinical Entity Classification + 自由文本）  
> **日期**：2026-07-15

---

## 1. 系统概览

### 1.1 任务定义

| 输入 | 输出 |
|------|------|
| 脑部 MRI 三模态体积扫描（T1 / T2 / FLAIR，各 128³ voxels） | 放射学报告文本（37 类病理实体分类 + 自由文本描述） |

### 1.2 总体架构

```
MRI 三模态 (128³×3)
       │
       ▼
┌──────────────────────────────┐
│   MRRCNN 编码器 × 3（独立模态） │  ← 3D CNN 金字塔 + StageAttention
│   各输出 40 tokens × 512 dim   │
└──────────────┬───────────────┘
               │
       ┌───────┴───────┐
       │  Linear(512→2048) × 3  │  ← 视觉→语言空间投影
       │  + mod_emb (3×1×2048)  │
       └───────┬───────┘
               │
       120 tokens × 2048 dim  (3 模态 × 40 tokens)
               │
               ▼
┌──────────────────────────────┐
│  Qwen2.5-3B-Instruct (LoRA)  │  ← 报告文本自回归生成
│  inputs_embeds 前缀拼接       │
└──────────────┬───────────────┘
               │
               ▼
     放射学报告文本输出
```

## 2. 编码器架构：MRRCNN

### 2.1 设计原则

V5 编码器从 V3/V4 的 UniFormer (CNN+Attention 混合) 回归到**纯 CNN 金字塔**设计，核心权衡：

| 维度 | V3/V4 (UniFormer) | **V5 (MRRCNN)** |
|------|-------------------|-----------------|
| 架构 | CNN + Attention 混合 | 纯 6 级 CNN 金字塔 |
| 参数量 | ~22M | **~28.8M**（3 模态独立） |
| Visual Tokens | 32 per mod | **40 per mod**（5 stage × grid³=8） |
| 显存 (batch=8) | ~29GB | **~9GB** |
| 每步计算 | — | **0.37s** |

### 2.2 网络结构

```
输入: (B, 1, 128, 128, 128)
│
├── Stem: Conv3d(1→32, k7, s2)                      → (B, 32, 64³)
│
├── Stage1: 2× ConvBlock(32→64, s2) + SE3D           → (B, 64, 32³)
├── Stage2: 2× ConvBlock(64→128, s2) + SE3D          → (B, 128, 16³)
├── Stage3: 3× ConvBlock(128→256, s2) + SE3D         → (B, 256, 8³)
├── Stage4: 2× ConvBlock(256→320, s2) + SE3D         → (B, 320, 4³)
├── Stage5: 2× ConvBlock(320→320, s1) + SE3D         → (B, 320, 4³)
│
├── Per-Stage Tokenization:
│   AdaptiveAvgPool3d(2³) → Conv3d→512 → flatten → + scale_emb
│   → 5 stages × 8 grid tokens = 40 tokens (512-dim)
│
└── StageAttention: 1-layer MultiheadAttention(4 heads) + MLP
    → (B, 40, 512)
```

### 2.3 关键组件

**ConvBlock**（残差卷积块）：

$$\text{out} = \text{ReLU}(\text{SE}(\text{BN}(\text{Conv}_2(\text{ReLU}(\text{BN}(\text{Conv}_1(x)))))) + \text{Ds}(x))$$

其中 Ds 为可选的 1×1×1 下采样卷积（stride=2），SE 为 3D 通道注意力模块。

**SE3D**（通道注意力）：

$$\text{SE}(x) = x \cdot \sigma(\text{Conv}_{1\times1\times1}(\text{ReLU}(\text{Conv}_{1\times1\times1}(\text{GAP}(x)))))$$

**StageAttention**（跨阶段自注意力）：

对 5 个 stage 共 40 个 token 做单层 MHA + MLP，借鉴 Transformer 的全局交互能力但计算量极小（仅 40 个 token）。

### 2.4 三模态编码

```python
class ReportingModelV5:
    t1_enc: MRRCNN       # T1 独立编码器
    t2_enc: MRRCNN       # T2 独立编码器
    flair_enc: MRRCNN    # FLAIR 独立编码器
    t1_proj: Linear(512, 2048)    # → LLM 空间
    t2_proj: Linear(512, 2048)
    flair_proj: Linear(512, 2048)
    mod_emb: (3, 1, 2048)         # 模态标识嵌入
```

三模态各自经过独立 MRRCNN 编码后投影到 Qwen 的 2048 维嵌入空间，加上模态嵌入后拼接为 `[B, 120, 2048]` 的 visual token 序列。

## 3. 三阶段训练策略

### 3.1 设计动机

直接将 CNN 编码器 + Qwen 端到端训练面临两个问题：
1. CNN 输出空间（512 dim）与 Qwen 嵌入空间（2048 dim）不兼容
2. Qwen 3B 的全量微调显存需求过高（> 80GB），且容易灾难性遗忘

因此采用三阶段递进式训练：先独立训练编码器，再对齐嵌入空间，最后轻量微调语言模型。

### 3.2 Phase 1：编码器对比学习

**目标**：使编码器学会提取有意义的视觉特征，不依赖任何外部模型。

**损失函数**（InfoNCE）：

$$\mathcal{L}_{\text{contra}} = -\frac{1}{|P|}\sum_{(i,j)\in P} \log\frac{\exp(s_{ij}/\tau)}{\sum_{k\notin P_i}\exp(s_{ik}/\tau)}$$

- 正样本对：同一患者的不同 MRI 模态（T1/T2/FLAIR 两两组合）
- 负样本：不同患者的任意模态
- 温度参数 $\tau = 0.07$
- 投影头：`mean_pool → Linear(512→512) → LayerNorm → ReLU → Linear(512→256) → L2_normalize`

**训练配置**：

| 参数 | 值 |
|------|------|
| 优化器 | AdamW (lr=3e-4, wd=1e-4) |
| 调度器 | CosineAnnealingLR (eta_min=lr×0.01) |
| Batch Size | 16 |
| Epochs | 5 |
| 梯度裁剪 | 1.0 |
| 冻结部分 | t1_proj, t2_proj, flair_proj |
| 数据量 | 88,985（全量）/ 4,587（batch27 验证） |

**显存优势**：Phase 1 不加载 Qwen，仅训练 CNN 编码器，显存仅 ~9GB（batch=8）。

### 3.3 Phase 2：投影层嵌入对齐

**目标**：使 encoder 输出的 visual tokens 的分布与 Qwen 文本嵌入空间对齐。

**损失函数**（Mean-Pooling MSE）：

$$\mathcal{L}_{\text{align}} = \left\|\frac{1}{120}\sum_{k=1}^{120} v_k - \frac{1}{|T|}\sum_{t\in T} \text{Emb}(w_t)\right\|^2$$

其中 $\{v_k\}_{k=1}^{120}$ 为 visual tokens，$\text{Emb}(w_t)$ 为 Qwen 的 token embedding（仅查表，不做 LLM 前向传播）。

**训练配置**：

| 参数 | 值 |
|------|------|
| 优化器 | AdamW (lr=1e-4, wd=1e-4) |
| Batch Size | 4 |
| Epochs | 3 |
| 冻结部分 | 三个 MRRCNN 编码器 |
| 训练部分 | t1_proj, t2_proj, flair_proj, mod_emb |
| 训练时长 | ~3 小时（loss: 0.074 → 0.0002） |

**已知局限**（详见根因分析 §8）：全局均值池化导致 120 个 token 收敛到同一向量，有效秩 ≈ 1。

### 3.4 Phase 3：Qwen LoRA 报告生成

**目标**：在冻结编码器的基础上，通过 LoRA 轻量微调 Qwen 实现报告生成。

**嵌入拼接方式**：

```python
inputs_embeds = cat([visual_tokens, text_tokens], dim=1)  # 视觉前缀
labels       = cat([IGNORE_INDEX × 120, target_ids], dim=1)  # 视觉位置忽略
```

**LoRA 配置**：

$$\Delta W = BA, \quad B \in \mathbb{R}^{d \times r}, \quad A \in \mathbb{R}^{r \times d}$$

| 参数 | 值 |
|------|------|
| LoRA rank r | 8 |
| LoRA alpha | 16 |
| Dropout | 0.05 |
| 目标模块 | q_proj, k_proj, v_proj, o_proj |
| LoRA 参数量 | ~6.2M（Qwen 3B 的 0.2%） |

**训练配置**：

| 参数 | 值 |
|------|------|
| 优化器 | AdamW (lr=5e-5, wd=0.01) |
| Batch Size | 8 |
| Epochs | 3 |
| AMP | 启用 (bfloat16) |
| 梯度检查点 | 启用 |
| 显存 | ~30GB |

## 4. 联邦学习（FedAvg）

### 4.1 问题

全量 88,985 样本在单卡上训练受 NAS I/O 瓶颈限制（batch16 下每步 ~35s，GPU 计算仅 0.3s），因此将数据按 batch 切分到 4 张 GPU 并行训练。

### 4.2 四步流程

```
Step 1: generate_init(seed=42) → 共享初始权重
              │
    ┌─────────┼─────────┬─────────┐
    ▼         ▼         ▼         ▼
  GPU0      GPU1      GPU2      GPU3
batch00-04 batch05-09 batch10-14 batch15-19
 13,298    17,758    13,374    44,555 样本
    │         │         │         │
    ▼         ▼         ▼         ▼
 各自 Phase 1 训练 5 epoch
    │         │         │         │
    └─────────┼─────────┴─────────┘
              │
Step 2: Head 对齐 — 对每个 MHA 头的排列找最优匹配
Step 3: FedAvg 加权平均 — 按样本量加权
Step 4: BN 重校准 — 全量数据 forward 更新 running stats
              │
              ▼
    merged_phase1.pt（最终共享编码器权重）
```

### 4.3 数学正确性保证

1. **相同初始化**：保证线性模式连通，避免 permutation invariance 导致的平均退化
2. **同患者不跨 GPU**：对比学习的 positive pair 完整性不受影响
3. **卷积线性性**：$\text{Conv}(\frac{W_1+W_2}{2}) = \frac{\text{Conv}(W_1) + \text{Conv}(W_2)}{2}$（无 bias 时精确成立）
4. **MHA Head 对齐**：枚举 4! = 24 种排列，选择 Frobenius 距离最小者
5. **BN 重校准**：避免子集 running_mean/running_var 偏差

## 8. 已知问题与改进方向

### 8.1 核心问题

Phase 3 评估 Clinical F1 接近 0，根本原因已定位为两个：

1. **Phase 1 对比学习目标与病变识别失配**：InfoNCE 优化的是患者区分（$I(X; \text{Patient})$），而非病变识别（$I(X; \text{Disease})$）。4500+ 位患者 vs 37 类病变的信息量差异使模型倾向于学习患者身份特征。

2. **Phase 2 Mean-Pooling 导致 Token 信息塌缩**：将 120 个 visual tokens 全局平均池化后与文本均值做 MSE 对齐，梯度被 $1/K$ 稀释。优化器隐式偏差使所有 token 趋同，有效秩 $\text{rank}(V) \approx 1$，LLM 实际收到的视觉信息等价于单个向量重复 120 次。

### 8.2 改进方向

| 方向 | 方案 | 预期效果 |
|------|------|----------|
| 分类预训练 | 用 37 类 Clinical Entity 标签做多标签分类替代对比学习 | Clinical F1 从 ~0 → 0.15–0.30 |
| Learnable Query Token | 用 Cross-Attention 从 120 tokens 提取 M=8 个 learnable query，替代 mean-pooling | 解决 token 塌缩 |
| 跳过 Phase 2 | Projector + LoRA 端到端联合训练 | 消除两阶段级联误差 |
| 全量数据 | 88,985 样本训练（已在 farm02 4 GPU 并跑） | encoder 特征质量 3–5× 提升 |
